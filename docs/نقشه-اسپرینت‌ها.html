Sprint 2 — آرشیو و جستجوی سازمانی

هدف: از «یک جلسه» به «حافظه سازمانی جلسات» برسیم.

قابلیتتوضیحفنی (پیشنهادی)
RAG چندجلسه‌ای پرسش درباره چند جلسه با فیلتر تاریخ، پروژه، شرکت‌کننده Chroma metadata: org_id, project, date؛ فیلتر در query
صفحه آرشیو لیست جلسات با جستجوی متنی روی عنوان/خلاصه SQLite FTS5 روی title, summary
برچسب و پروژه دسته‌بندی جلسات (مثلاً standup، planning، review) ستون tags, project_key در SQLite
خروجی خلاصه Export به Markdown / PDF برای اشتراک با مدیر قالب Astro یا WeasyPrint از همان MeetingAnalysis
مقایسه جلسات «تصمیمات sprint قبل vs الان» — agent با context چند meeting_id compare_agent + top-k از هر جلسه

معیار پذیرش Sprint 2

  • ۳ جلسه ingest شده؛ یک سوال cross-meeting با citation از ≥۲ جلسه
  • جستجو در آرشیو زیر ۵۰۰ms (تا ~۱۰۰ جلسه)
  • Export خلاصه یک جلسه بدون خطای RTL

Sprint 3 — Jira و گردش کار عملیاتی

هدف: تسک‌ها واقعاً به افراد در Jira برسند، نه فقط issue ناشناس.

قابلیتتوضیحفنی
نگاشت assignee «سارا» در transcript → accountId Jira جدول speaker_jira_map در SQLite؛ UI تنظیمات
jira_agent پیش‌نمایش، dedup، پیشنهاد epic/link Pydantic AI tools: search_existing_issues, preview_create
OAuth Jira جایگزین API token شخصی — مناسب تیم Atlassian OAuth 2.0؛ ذخیره refresh token رمزنگاری‌شده
ویرایش قبل از ارسال کاربر تسک‌ها را در UI ویرایش کند، بعد bulk create state موقت؛ POST فقط پس از تأیید
وضعیت همگام نمایش issue key و لینک Jira در صفحه جلسه ستون jira_keys_json per task
قالب issue فیلدهای سفارشی پروژه (story points، component) map از JIRA_PROJECT_KEY + config YAML

Sprint 4 — کاربران، فضای کار، و تقویم

هدف: چندکاربره واقعی و اتصال به ابزارهای روزمره (بدون صوت).

قابلیتتوضیحفنی
احراز هویت ورود با ایمیل/رمز یا SSO سازمانی JWT session؛ یا OIDC (Keycloak / Google Workspace)
فضای کار (workspace) هر سازمان داده جدا — جلسات، نقشه speaker، Jira org_id در همه جداول؛ middleware tenancy
نقش‌ها (RBAC) admin · editor · viewer — چه کسی Jira create کند جدول memberships
تقویم (metadata) import عنوان/زمان/شرکت‌کنندگان از Google Calendar — transcript همچنان paste/upload Calendar API read-only؛ پیش‌پر کردن فرم جلسه
ایمیل خلاصه ارسال خودکار خلاصه + لینک جلسه به شرکت‌کنندگان SMTP یا Resend/Cloudflare Email؛ قالب HTML RTL
اعلان Slack/Teams پیام کوتاه: N تسک جدید + لینک Incoming webhook

Sprint 5 — مقیاس، کیفیت، استقرار

هدف: آماده دمو سازمانی و پایش در production.

قابلیتتوضیحفنی
PostgreSQL جایگزین SQLite برای چند worker و backup SQLAlchemy + Alembic migration از schema فعلی
Vector در مقیاس هزاران chunk بدون کندی pgvector یا Chroma Cloud؛ retention policy
صف ingest تحلیل جلسه طولانی در background Redis + worker (Celery / ARQ)؛ UI polling وضعیت
Observability ردیابی latency Gemini، خطای Jira OpenTelemetry + structured logs؛ dashboard
استقرار Docker Compose؛ اختیاری Cloud Run / VPS frontend + API + Postgres در compose
eval و prompt مجموعه golden transcript؛ regression کیفیت خلاصه pytest + امتیاز LLM-judge یا human rubric
محدودیت نرخ جلوگیری از سوءاستفاده API rate limit per org؛ cache embedding
