۱. راهنمای برگزارکننده — جلسات کارآمدتر در آینده

پس از تحلیل هر جلسه، سیستم به برگزارکننده (facilitator) پیشنهاد عملی می‌دهد: چه چیزی خوب بود، کجا وقت هدر رفت، و جلسه بعدی چطور کوتاه‌تر و مؤثرتر برگزار شود. 

sequenceDiagram
    participant O as برگزارکننده
    participant UI as Astro
    participant API as FastAPI
    participant FA as facilitation_agent
    participant ST as آمار مشارکت

    O->>UI: پایان جلسه · ingest
    API->>ST: speaker stats · مدت · تعداد تصمیم/تسک
    API->>FA: transcript + stats + نوع جلسه
    FA-->>UI: FacilitationReport
    UI-->>O: راهنمای فارسی + چک‌لیست جلسه بعد

چک‌لیست دستی (امروز — بدون AI)

قبل از جلسه

  • هدف یک‌خطی و خروجی مورد انتظار (تصمیم / تسک / اطلاع‌رسانی) مشخص باشد
  • دستور کار با زمان تقریبی هر بند (timebox) ارسال شود
  • فقط افراد ضروری دعوت شوند — نقش هر نفر روشن باشد
  • اسناد مرجع (SOW، قرارداد) از قبل در دسترس باشد

حین جلسه

  • تصمیمات را با جمله «تصمیم گرفتیم که…» و مسئول اعلام کنید
  • بحث‌های off-topic را به «پارکینگ lot» بسپارید
  • هر ۱۵–۲۰ دقیقه خلاصه ۳۰ ثانیه‌ای از پیشرفت

بعد از جلسه

  • خلاصه + تسک‌ها ظرف ۲۴ ساعت منتشر شود
  • جلسه follow-up فقط در صورت تسک باز برنامه‌ریزی شود

خروجی پیشنهادی facilitation_agent

فیلدتوضیح
what_went_well۲–۳ نکته مثبت (مثلاً تصمیمات شفاف، مشارکت متعادل)
improvementsپیشنهاد مشخص (مثلاً «بخش X ۴۰٪ زمان را گرفت بدون تصمیم»)
next_meeting_agendaپیش‌نویس دستور کار جلسه بعد از تسک‌های باز
timebox_suggestionپیشنهاد مدت کوتاه‌تر یا format متفاوت (async update)
facilitator_scoreامتیاز ۱–۵ بر اساس rubric (اختیاری)

API: GET /api/meetings/{id}/facilitation — ✅ پیاده‌سازی شده · تب «راهنمای جلسه» در UI

۲. آپلود اسناد (SOW، قرارداد) — هم‌راستایی با scope

سازمان می‌تواند Statement of Work، قرارداد، یا RFP را آپلود کند. پس از هر جلسه، alignment_agent تصمیمات و تعهدات جلسه را با بندهای سند مقایسه می‌کند: in-scope، out-of-scope، یا نیاز به تأیید حقوقی/مدیر.

۳. تحلیل احساسات (Sentiment) — لحن هر گوینده

روی متن transcript (نه صوت خام)، برای هر گوینده و هر بخش جلسه: لحن کلی (مثبت / خنثی / منفی)، ناهماهنگی (مثلاً کلمات مثبت با نگرانی پنهان)، و روند احساس در طول جلسه.

خروجی SpeakerSentimentProfile

فیلدمعنی

API پیشنهادی: GET /api/meetings/{id}/sentiment — تب جدید در صفحه جلسه کنار خلاصه/تسک/RAG.

اخلاق و حریم خصوصی: تحلیل sentiment برای مدیریت تیم حساس است. فقط admin/facilitator باید ببیند؛ opt-in سازمانی؛ بدون برچسب‌گذاری منفی در گزارش‌های عمومی. در UI: «سیگنال برای گفت‌وگوی follow-up» نه «قضاوت شخصیتی».
